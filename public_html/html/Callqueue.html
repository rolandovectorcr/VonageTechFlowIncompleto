Call Queue troubleshooting

Step 1

Call the extension directly. If the call does not ring the extension, troubleshoot the extension. If extension rings, confirm the extension has been configured properly.

Example: If the call queue is set to assign calls to agents based on tier, make sure the extension is in the right tier.

Manually log the agent out and then back into the queue and Check the Status.

Delayed: The issue is probably with the internet connection.

Available: Make a test call to see if issue resolved.

Step 2

Remove the extension from the Call Queue and save. Add the extension back into the Call Queue and save. Place a test call to see if issue resolved.

Go to the Directory to determine if problem is a stuck call.

Step 3

Proceed to rebuild the Call Queue.

CAUTION: BEFORE PROCEEDING SAVE THE RECORDINGS AND THE SETTINGS THAT ARE CONFIGURED ON CALL QUEUE.

Add a new Call Queue using HDAP and configure it with the simplest settings, proceed to save and test.

Were you able to save the new Call Queue?

No: Remove the new Call Group and report the issue to Sup/TL before escalating it. *How to escalate issue.

Yes: Configure the new Call Queue with the same settings as the original. Save and test.

Did this resolve the issue?

Yes: Go main menu

No -> Remove the new Call Queue and report the issue to Sup/TL before escalating it.*How to escalate issue.
